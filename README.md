# 🚀 DSA Logic Building Journey with Python

> **"Before solving Data Structures & Algorithms, learn how to think like a programmer."**

This repository is my personal **DSA Preparation Journey**, where I build programming logic from scratch before diving into advanced Data Structures and Algorithms.

Instead of memorizing solutions, this repository focuses on developing **problem-solving ability**, **logical thinking**, and **clean coding practices**.

---

# 🎯 Goal

Most beginners jump directly into DSA and struggle because their programming logic is weak.

This repository follows a structured roadmap:

```
Programming Logic
        ↓
Loop Mastery
        ↓
Pattern Thinking
        ↓
Problem Solving
        ↓
Data Structures
        ↓
Algorithms
        ↓
Coding Interviews
```

The objective is to become a strong problem solver rather than someone who only remembers solutions.

# 📚 Repository Roadmap

## ✅ Phase 1 — While Loop (Logic Foundation)

Build the fundamentals of programming using **while loops**.

Topics include:

* Number printing
* Reverse counting
* Even & Odd numbers
* Multiplication tables
* Sum of numbers
* Factorial
* Digits extraction
* Reverse numbers
* Palindrome numbers
* Armstrong numbers
* Prime numbers
* Fibonacci series
* Pattern-based thinking

Focus:

* Loop understanding
* Variable manipulation
* Logical reasoning

---

## ✅ Phase 2 — Do-While Thinking & Control Flow

Python doesn't have a native `do-while` loop, so these exercises simulate the same execution style using `while True`.

Topics include:

* Execute-at-least-once logic
* Infinite loops
* Break statement
* Continue statement
* Input validation
* Sentinel-controlled loops
* Menu-driven programs
* Control flow mastery

Focus:

* Real-world programming logic
* Decision making
* Flow control

---

## ✅ Phase 3 — Clean Coding & FOR Loop Optimization

Convert previous logic into cleaner and more optimized implementations using Python's `for` loop.

Topics include:

* Range optimization
* Cleaner syntax
* Reduced code complexity
* Better readability
* Efficient iteration
* Pythonic coding style

Focus:

* Writing production-quality code
* Code optimization
* Readability
